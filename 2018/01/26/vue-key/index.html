为什么v-for要搭配key使用

Posted on 2018-01-26 | In vue

vue推荐: 使用v-for的时候,添加key属性
为什么呢?

  • 因为vue有 “就地复用”策略:
  • 如果不加key的话,在数组的前面添加,一个对象,文本框里的内容会根据 索引 实现一个就地复用
  • 所以v-for的时候,添加一个属性 key
  • 一般使用 id 而不是index , 因为顺序变了使用index就不准确了
  • :key = ‘item.id’
    以后使用v-for的时候请务必加上key值
